# 🌍 Overview

A Django application processes every user request in several steps.

```
Browser
   │
   ▼
urls.py
   │
   ▼
views.py
   │
   ▼
models.py
   │
   ▼
Database
   │
   ▼
Template
   │
   ▼

```

Each file has its own responsibility.

---

# 📌 Django Project Structure

| File | Purpose |
|------|---------|
| `urls.py` | Maps URLs to views |
| `views.py` | Handles HTTP requests and responses |
| `models.py` | Defines database structure |
| `templates/` | Generates HTML pages |

---

# 1️⃣ URLs (`urls.py`)

## Purpose

- Receives HTTP requests.
- Matches URLs.
- Sends requests to the correct View.

### Example

```python
from django.urls import path
from . import views

urlpatterns = [
    path("books/<int:id>/", views.book_detail),
]
```

### Key Points

- `path()` creates URL routes.
- `<int:id>` captures an integer from the URL.
- Calls the appropriate view function.

---

# 2️⃣ Views (`views.py`)

## Purpose

- Receives HTTP requests.
- Processes business logic.
- Retrieves data from models.
- Returns an HTTP response.

### Example

```python
from django.http import HttpResponse

def index(request):
    return HttpResponse("Hello from Django!")
```

### Key Points

- Every view receives a `request`.
- Every view returns a `response`.
- Views act as the brain of the application.

---

# 3️⃣ Models (`models.py`)

## Purpose

Models represent database tables.

Instead of writing SQL manually, Django uses Python classes.

### Example

```python
from django.db import models

class Team(models.Model):
    team_name = models.CharField(max_length=40)
```

### Key Points

- One Model = One Database Table
- One Object = One Database Record
- Django automatically creates SQL behind the scenes.

---

# 4️⃣ Querying the Database

Views retrieve data using Models.

### Example

```python
from .models import Team

teams = Team.objects.filter(team_level="U09")
```

### Key Points

- No raw SQL required.
- Uses Django ORM.
- Easy to read and maintain.

---

# 5️⃣ Templates

## Purpose

Templates generate dynamic HTML pages.

### Example

```html
<h1>{{ team.team_name }}</h1>
```

### Key Points

- `{{ }}` displays variables.
- `{% %}` contains template logic.
- Keeps HTML separate from Python code.

# 🏗️ Django MVT Architecture

| Component | Responsibility |
|-----------|----------------|
| **Model** | Database and data |
| **View** | Business logic |
| **Template** | User Interface (HTML) |

|------------|-------------|-------------------|
| `urls.py` | Routes requests | Receptionist |

---

# 💡 Key Takeaways

- Django separates code into multiple files.
- `urls.py` decides **where** requests go.
- `views.py` decides **what** happens.
- `models.py` manages the database.
- Templates create HTML pages.
- Django follows the **MVT (Model–View–Template)** architecture.
- Separation of responsibilities makes projects easier to build and maintain.
